# How to Study This Project on GitHub

You do not need Obsidian or another note application. Open the
[Defense Study Guide](README.md) in a browser and follow its study order.

## Useful GitHub features

- Click any Markdown link to move between lessons.
- Open the [single Java source](../src/TerminalSimulation.java) beside a lesson
  to compare the explanation with the actual implementation.
- Use the file outline to jump between headings.
- Mermaid diagrams render directly on GitHub.
- Expand the hidden answers in Rapid Recall and Trace Drills only after answering.
- Use GitHub's search box to find a class or method across the repository.

## Recommended browser layout

1. Open the relevant lesson in one tab.
2. Open `src/TerminalSimulation.java` in a second tab.
3. Search the source for the method named by the lesson.
4. Explain what enters the method, what changes, and what happens next.

## Practice with an AI assistant

Copy the prompt from [Mock Defense](05%20-%20Practice/Mock%20Defense.md) into the
assistant you use. Tell it to inspect the GitHub repository and ask one question
at a time. Always compare its correction against the actual Java source.

> [!WARNING]
> An assistant can help you practice, but each group member must be able to
> explain the code without reading generated answers during the defense.
